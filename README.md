# python-stack-progressive-uc
A progressive Python Stack project implementing 10 advanced use cases, where each new UC extends the previous one by adding a new feature, optimization, or real-world stack application.

## Use Cases

### UC1 – Design Generic Stack ADT
Create a reusable Stack Abstract Data Type with encapsulation, validation, and support for multiple data types.

### UC2 – Implement Dynamic Stack with Capacity Control
Extend UC1 by adding dynamic growth, optional maximum capacity, and overflow/underflow exception handling.

### UC3 – Build Stack Using Linked List
Extend previous functionality by implementing the same stack behavior using linked list instead of list-based storage.

### UC4 – Implement Multiple Stacks in a Single Array
Extend stack design to support two or more stacks in one shared array for memory-efficient storage.

### UC5 – Develop Advanced Parentheses and Symbol Validator
Use the stack to validate balanced symbols and nested expressions with multiple bracket types.

### UC6 – Create Infix to Postfix and Prefix Conversion Engine
Extend symbol-handling logic to convert mathematical expressions using operator precedence and associativity.

### UC7 – Build Expression Evaluation System
Extend UC6 by evaluating postfix and prefix expressions with proper operand and operator handling.

### UC8 – Implement Undo/Redo State Management System
Extend stack usage to real-world application flow by managing undo and redo operations using two stacks.

### UC9 – Design Special Stack with O(1) Min and Max Retrieval
Extend basic stack behavior by maintaining additional internal structures for constant-time minimum and maximum retrieval.

### UC10 – Solve Advanced Stack Problems with Testing and Performance Analysis
Extend the stack system to solve advanced problems like next greater element, stock span, and largest rectangle in histogram, with unit testing and complexity analysis.
